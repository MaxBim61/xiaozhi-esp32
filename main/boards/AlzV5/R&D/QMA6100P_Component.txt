 Task1:

Нам нужно создать компонент драйвера для QMA6100P, который будет использоваться в проекте Xiaozhi-esp32. Мы должны учесть схему подключения из AlzV5-HSI.docx:
 пины 
 GPIO19 (INT1),
 GPIO20 (INT2),
 GPIO40 (SDA),
 GPIO41 (SCL).
 Акселерометр подключен по I2C. Также нужно реализовать функцию wake-on-motion: возможно, настроить прерывание по движению, чтобы выводить ESP32 из глубокого сна.
 Настроить акселерометр: установить режим измерения, диапазон, выходную частоту.
Настроить детектор движения (any-motion) для wake-up.
Настроить прерывание на INT1 (GPIO19) для пробуждения из глубокого сна.
Интегрировать с системой управления питанием Xiaozhi (скорее всего, в проекте есть менеджер сна/пробуждения).
Plan:
Создать драйвер QMA6100P как отдельный компонент в проекте (в папке components/qma6100p или в main/boards/AlzV5_esp32).
Реализовать базовую инициализацию I2C, чтение/запись регистров.
также нужно учитывать, что проект Xiaozhi-esp32 уже может иметь какую-то структуру для добавления новых плат. Возможно, нужно дополнить AlzV5.cc вызовом инициализации акселерометра и настройкой wake-up 
кселерометр QMA6100P имеет I2C адрес (вероятно, 0x12 или 0x13? Нужно уточнить в даташите. По умолчанию адрес QMA6100P - 0x12 (если ADDR подключен к GND) или 0x13 (к VDDIO). На схеме не указано подключение адресного пина. Предположим, что ADDR на GND, адрес 0x12. Можно добавить возможность конфигурации.

Также нужно определить регистры для настройки any-motion: регистры 0x09 (Motion_Config), 0x0A (Motion_Threshold) и т.д. Ссылаться на даташит.

Реализовать  с использованием I2C master из ESP-IDF, настройку прерывания через gpio_isr_handler, и настройку deep sleep wake-up через gpio_wakeup_enable.

Дополнительные улучшения
1, Сохраняйте состояние акселерометра в RTC-памяти, чтобы после пробуждения не перенастраивать его полностью.
2, Для более точного детектора можно настроить порог и длительность в соответствии с требованиями.


 Task2:
 
Дополнить предыдущую реализацию драйвера QMA6100P и логику управления для обработки события "одинарный удар" (single tap) с порогом 1.25g и использованием INT2 для пробуждения, а также реализовать логику таймаута бездействия для вызова emergency().

Необходимо:

Настроить QMA6100P для детектирования одинарного удара (single tap) на INT2. Уточнить по даташиту регистры для tap/double-tap.

Включить прерывание по tap на INT2, настроить порог, длительность, и возможно другие параметры (латентность, окно).

В основном коде после пробуждения по INT2 (определить причину пробуждения) выполнить определенное действие (например, через спикер спросить "Что случилось?" и попытаться войти в диалог с Xiaozhi).

После пробуждения запустить таймер на 1 минуту ожидания голосового контакта и/или движений (INT1 - any-motion). Если за минуту нет голосового ввода и нет any-motion, вызвать emergency().

Для этого потребуется:

Доработка драйвера QMA6100P с поддержкой tap/double-tap.

Настройка обоих прерываний: any-motion на INT1, tap на INT2.

Интеграция с системой энергосбережения Xiaozhi.

Реализация логики таймера и обработки событий.

Также необходимо учесть, что INT1 используется для any-motion (пробуждение по движению), а INT2 - для tap (пробуждение по удару). Но возможно, что в глубоком сне оба прерывания могут быть использованы для пробуждения, но ESP32-S3 может просыпаться только по одному источнику? На самом деле можно настроить пробуждение по нескольким GPIO одновременно (esp_sleep_enable_gpio_wakeup позволяет задать маску пинов с уровнем/фронтом). Так что можно разрешить пробуждение по обоим пинам.
